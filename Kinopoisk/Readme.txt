   Курсовой проект, БД Kinopoisk.

1. О проекте.
БД Kinopoisk отображает данные, представленные на сайте https://www.kinopoisk.ru. 
Основным объектом вокруг которого строится вся работа – страница детализации выбранного фильма, например, https://www.kinopoisk.ru/film/349. 
В БД хранятся данные фильма, список лиц, задействованных в производстве фильма, данные этих лиц, а также рецензии зрителей на данный фильм.

2. О дизайне.
Некоторые справочные данные организованы в виде set (movies.genre), некоторые в виде таблиц (countries, mpaa_ratings, person_statuses). Целесообразность использования 
set в данном случае сомнительна, так как на сайте скорее всего где-то есть выпадающий список с данными genre (жанры) и заполнятся он должен из справочной таблицы. 
В противном случае данные для выпадающего списка (точно такие же) придётся держать в другом месте, а это уже дублирование что не есть правильно. 
Но здесь сделано set для демонстрации усвоения материала.
С другой стороны set в качестве меры денормализации имеет смысл в плане производительности, если такая денормализация имеет место (set и enum это скрытые таблицы 
или check constraint?).

3. О скриптах выборок данных.
Данные выбираются в таком количестве и качестве, как они отображаются на страницах сайта. Некоторые моменты упрощены, например иконка страны заменена на название 
по понятным причинам. Картинки хранятся в BLOB, и они так же не попадают в выборку.
